Apache ECharts v5 — Example Charts

Evaluation file — shows ECharts equivalents of the D3.js charts used in Sales, Purchasing & Inventory Dashboards, with built-in animations, DataZoom, Toolbox, and an SVG ↔ Canvas renderer toggle on every chart.

SVG Renderer Scalable Vector Graphics

How it works

ECharts draws each chart element as an individual DOM node ( <path>, <rect>, <text> …) inside an <svg> tag embedded in the page.

✓ Crisp at any zoom or print scale — charts look perfect when a user zooms the browser or prints the page.

✓ Accessibility — screen readers can traverse the DOM tree; tooltips remain keyboard-accessible.

✓ Copy-as-image — most browsers let you right-click an <svg> and save it directly.

✓ CSS styling — you can target chart elements with your existing Tailwind / CSS classes.

✗ Slower with very large datasets — each data point is a DOM node; performance degrades above ~10 000 points.

Recommendation for FB BI Reports: Use SVG. Your datasets are small (monthly aggregates, top-N lists) and print quality / zoom crispness matters more than raw rendering speed.

Canvas Renderer HTML5 2D Bitmap

How it works

ECharts paints directly onto an HTML <canvas> element using the 2D drawing API — a single flat bitmap, no individual DOM nodes per data point.

✓ High-volume data — handles 100 000+ points without slowing the browser; ideal for real-time charts or scatter plots with thousands of individual points.

✓ Complex animations — frame-by-frame rendering is faster when every pixel changes each frame.

✗ Blurry when zoomed — it's a bitmap; zooming the browser or printing at high DPI produces a pixelated/blurry image unless you increase the devicePixelRatio manually.

✗ Not accessible — the canvas is opaque to screen readers.

✗ No CSS targeting — you can't select individual bars or lines with CSS.

When to use Canvas: Only if you add a chart with >10 000 individual rendered points (e.g. a raw transaction scatter plot). Switch that chart instance to Canvas; leave all others on SVG.

DataZoom

ECharts ships a slider (the bar at the bottom) and an inside zoom (scroll-wheel / two-finger pinch on the plot area) with zero extra code. With D3 you would have to implement brush, zoom and pan by hand (~150 lines). Both zoom types always stay in sync.

Toolbox

The icons in the top-right corner are the built-in toolbox component. Save image exports a PNG (SVG renderer) or a PNG screenshot (Canvas). Data view opens the raw numbers in a text table so users can copy-paste into Excel. Restore resets any zoom. All built-in, no extra code.

Animations

On first render the bars rise from the baseline and the line draws itself left-to-right. ECharts also animates on data updates (e.g. when a filter changes). The easing, duration and staggering are all configurable or can be disabled in one line (animation: false).

Scrollable legend vs D3 custom legend

The existing D3 donut manually builds an HTML legend with absolute positioning (~60 lines of JS). ECharts provides a scrollable legend component that handles overflow automatically. Clicking a legend item toggles the slice with a smooth animation — no extra event listeners needed.

Nightingale Rose

ECharts can switch the same data between donut, pie, and Nightingale Rose (where radius encodes value, not just arc angle) with a single option change. Try the buttons above — the transition is fully animated. The D3 equivalent would require a complete redraw with a different layout function.

Value labels & colour mapping

Each bar shows its currency value as a built-in label (no extra SVG text elements). Bar colour is mapped to margin % — green for healthy margins (>30%), amber for medium (15–30%), red for low (<15%). In D3 you would calculate this in JavaScript and manually set fill per bar; here it's a one-liner inside the itemStyle callback.

DataZoom for long lists

If there are 20+ salesperson rows the slider at the bottom lets the user scroll without the chart becoming unreadably cramped. The existing D3 charts rely on a fixed height calculation that gets unwieldy with large datasets. ECharts DataZoom handles this cleanly at the component level.
